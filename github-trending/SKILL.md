# GitHub Trending Monitor

Fetches and analyzes GitHub Trending repositories for daily, weekly, and monthly timeframes.

## Usage

```
# Fetch daily trending
Use the skill with: since=daily (default)

# Fetch weekly trending  
Use the skill with: since=weekly

# Fetch monthly trending
Use the skill with: since=monthly
```

## Parameters

- `since`: Time range - one of `daily`, `weekly`, `monthly` (default: `daily`)

## Implementation

### Step 1: Fetch GitHub Trending Page

Use curl with proper headers to fetch the trending page:

```bash
curl -s "https://github.com/trending?since=${SINCE}" \
  -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36' \
  -H 'Accept: text/html,application/xhtml+xml' \
  --connect-timeout 30 \
  --max-time 120
```

### Step 2: Parse HTML and Extract Projects

The HTML structure for each project:
- Container: `<article class="Box-row">...</article>`
- Project link: `<h2 class="h3 lh-condensed"><a href="/owner/repo">`
- Owner: `<span class="text-normal">owner /</span>`
- Description: `<p class="col-9 color-fg-muted">description</p>`
- Language: `<span itemprop="programmingLanguage">Language</span>`
- Stars today: Pattern `([0-9,]+) stars today`

Use this Python code for parsing:

```python
import re

def parse_trending(html):
    articles = re.split(r'<article class="Box-row">', html)
    projects = []
    
    for article in articles[1:]:
        # Extract project link
        link_match = re.search(r'<h2 class="h3 lh-condensed">.*?href="/([^/]+)/([^"/]+)"', article, re.DOTALL)
        if link_match:
            owner, repo = link_match.groups()
            
            # Extract description
            desc_match = re.search(r'<p class="col-9[^>]*>([^<]+)', article)
            desc = desc_match.group(1).strip() if desc_match else ""
            
            # Extract language
            lang_match = re.search(r'itemprop="programmingLanguage">([^<]+)<', article)
            lang = lang_match.group(1).strip() if lang_match else ""
            
            # Extract stars today
            star_match = re.search(r'([0-9,]+)\s*stars?\s*today', article, re.IGNORECASE)
            stars_today = star_match.group(1).strip() if star_match else ""
            
            # Extract stars this week/month
            stars_period_match = re.search(r'([0-9,]+)\s*stars?\s*this\s*(week|month)', article, re.IGNORECASE)
            stars_period = stars_period_match.group(1).strip() if stars_period_match else ""
            
            projects.append({
                "owner": owner,
                "repo": repo,
                "url": f"https://github.com/{owner}/{repo}",
                "desc": desc,
                "lang": lang,
                "stars": stars_today or stars_period
            })
    
    return projects
```

### Step 3: Analyze Projects

For each project, provide:
1. **Project Name**: owner/repo
2. **URL**: https://github.com/owner/repo
3. **Language**: Programming language
4. **Stars**: Stars gained in the period
5. **Description**: Brief description
6. **Analysis**: AI-generated summary of what the project does and why it's trending

## Pitfalls

1. **Rate Limiting**: GitHub may rate-limit requests. Use proper delays between requests.
2. **HTML Changes**: GitHub may update their HTML structure. The regex patterns may need updates.
3. **Empty Results**: If no projects found, check if HTML structure changed.
4. **Encoding**: Handle HTML entities properly (e.g., `&amp;` -> `&`).
5. **Network Timeout**: Use appropriate timeouts; GitHub pages can be slow.

## Cron Integration

This skill works with Hermes cron jobs for automated monitoring. Three jobs have been created:

### Scheduled Jobs

| Job Name | Job ID | Schedule | Timeframe | Deliver |
|----------|--------|----------|-----------|---------|
| GitHub Trending 每日推送 | `222900afa555` | `0 9 * * *` | Daily at 09:00 | Feishu |
| GitHub Trending 每周推送 | `7e695d30fd62` | `0 9 * * 6` | Every Saturday at 09:00 | Feishu |
| GitHub Trending 每月推送 | `b71f5fb40b11` | `0 9 * * 6` | First Saturday of month at 09:00 | Feishu |

### Feishu Delivery Format

**重要**: 推送到飞书时，需要包含每个项目的详细描述（100字左右），格式如下：

```markdown
📊 GitHub Trending 每日热门 (2026-04-27)

🔥 今日 TOP 13 项目:

1️⃣ owner/repo ⭐ X,XXX
🔗 https://github.com/owner/repo
💡 详细描述项目功能、特点和使用场景，约100字...
...
---
数据来源: GitHub Trending | 采集时间: 2026-04-27
```

每个项目应包含：
- 排名和项目名
- 今日 stars 数量
- 仓库链接
- 详细功能描述（约100字）

### Cron Deliver Target

所有 GitHub Trending cronjob 应设置 `deliver: feishu:oc_09adea3128fe7a2b10a48188d2dd69f8`

### Scripts

- `fetch_trending.py` - Main script to fetch and parse trending data
  - Usage: `python3 fetch_trending.py --period [daily|weekly|monthly]`
  - Returns JSON array of projects
  
- `check_first_saturday.py` - Validates if today is the first Saturday of the month
  - Exit code 0 = first Saturday
  - Exit code 1 = not first Saturday
